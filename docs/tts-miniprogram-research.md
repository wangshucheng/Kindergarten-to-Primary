# 微信小程序中文 TTS 方案调研报告

> 信息获取日期：2026-07-19。价格/额度类信息时效性强，落地前请以官网最新页面复核。
> 调研方式：三条路线分别查阅一手官方文档（developers.weixin.qq.com / cloud.tencent.com / help.aliyun.com / xfyun.cn），免 Key 接口部分为当日实测。

## 0. 结论速览（TL;DR）

| 路线 | 可行性 | 长期成本 | 最大障碍 |
|---|---|---|---|
| ①a WechatSI 同声传译插件 | 视主体而定 | **0 元** | 个人主体基本不可用；单次限 50 字；音频链接 3 小时过期；无童声 |
| ①b 云开发 + 云函数中转 | ✅ 可行 | 19.9 元/月起 | 免费体验环境在小程序发布后第 15 天到期；只是中转层，音频仍需上游 TTS |
| ② 第三方云服务（腾讯/阿里/讯飞） | ✅ 可行 | **构建期一次性 0 元**（免费额度内）；运行时在线合成需叠加 ①b | 免费额度 3 个月过期、限领一次 |
| ③ 免 Key 直连接口 | ❌ 不可行（运行时） | 0 元 | 实测唯一存活的有道 dictvoice 中文限 ≤5 字；同类接口已全部失效；随时可能被封 |

**推荐方案（首选）**：**「云服务免费额度 + 构建期离线预生成」**——用腾讯云 TTS 的 800 万字符免费包（3 个月有效），在构建期把全部中文语料一次性合成 MP3 上传微信云存储，复用现有三级缓存架构。**运行时零成本、零密钥风险、零新增云服务依赖**，现有代码改动极小。

**演进兜底**：若未来接入动态生成的随机题干等运行时新文本，再加「云函数在线合成」兜底（成本 19.9 元/月起）。

## 1. 背景与需求

### 1.1 缺口

小程序端目前仅有 504 个英文单词的预生成音频；中文内容朗读被静默跳过（`miniprogram/src/platform/tts.ts:135`，`MINIPROGRAM_MIGRATION.md` 风险表「未预生成的文本无法朗读」）。当初放弃 WechatSI 插件的原因是类目/主体限制（`tts.ts:8` 注释）。

### 1.2 中文语料盘点（2026-07-19 实测统计）

| 内容 | 来源 | 规模 | 动态性 |
|---|---|---|---|
| 古诗文 | `miniprogram/src/games/poetry/poems.ts` | 18 首、72 行、正文约 480 字（含标题作者朗读约 700 字） | 静态 |
| 汉字 | `miniprogram/src/data/hanzi*.json` | 约 911 个中文串、约 1200 字 | 静态 |
| 拼音 | `miniprogram/src/data/pinyin*.json` | 约 349 个中文串、约 500 字 | 静态 |
| 应用题 | `wordProblemLogic.ts:29` | **恰好 8 条**静态题干、约 200 字 | 静态 |
| 几何题干 | `geometryLogic.ts` | 有限模板 × 有限图形名组合，可全枚举，约百级串 | 组合有限 |
| 固定引导语 | 各游戏组件（"听一听，选出正确的拼音！"等） | 数十条短句 | 静态 |

**关键结论：小程序端全部待朗读中文是构建期已知的有限集合，去重后总字符量 < 1 万**，腾讯免费包（800 万字符）约为其 1000 倍，离线预生成可 100% 覆盖需求。

### 1.3 硬性约束

- 免费或低成本；面向 5~7 岁儿童，不采集个人信息，不接第三方统计/广告 SDK；
- API 密钥不能放进小程序前端包；
- 优先复用现有 `TtsBackend` 接口与三级缓存（L1 内存 → L2 本地文件 → L3 云存储下载）；
- 项目已有微信云开发环境 `cloud1-d5g2o67hb3101ab8a`（`miniprogram/src/cloud-config.ts:24`）。

## 2. 路线一：微信官方能力

### 2.1 WechatSI（微信同声传译）插件

- **原理**：纯前端插件，无需服务端。公众平台添加插件（AppID `wx069ba97219f66d99`）→ `app.json` 声明 → `requirePlugin('WechatSI')` → `plugin.textToSpeech({lang:'zh_CN', content, success})`，回调返回音频 URL，直接喂 `wx.createInnerAudioContext()`。基础库 ≥ 1.9.6。
- **资费**：免费。配额：语音合成 **100 次/分钟、2 万次/天**（超额邮件申请 wetranslate@tencent.com）。来源：[官方文档·配额说明](https://developers.weixin.qq.com/miniprogram/dev/platform-capabilities/extended/translator.html)。
- **限制**：
  - **主体（致命）**：仅对企业等非个人主体开放，个人主体在插件管理中搜索不到也无法按 ID 添加（[微信开放社区官方回答，2026-05](https://developers.weixin.qq.com/community/develop/doc/00044489768378b580150936b61800?jumpto=comment&commentid=0008ea0a76c920a583157a917644)）；本项目代码注释已实证踩坑（`tts.ts:8`）。
  - **单次 content 限 50 字符**（官方文档原文）——古诗长文须拆段多次调用再拼接播放；8 条应用题题干均 < 50 字，可用。
  - 音频 URL **3 小时过期**（`expired_time`），不能直接当长期缓存源。
- **合规**：官方插件，只上行待合成文本，不采集个人信息，与儿童合规兼容。
- **稳定性**：微信官方维护（2026-04 仍有 0.3.6 版实战记录），无停用先例；但音色/语速均不可调，能力不可定制。
- **接入复杂度**：低。返回 URL 直接播放（不走域名白名单）；但因 3 小时过期，长期缓存需合成后立即 `wx.downloadFile` 落盘（音频域名未在官方文档列出，**未能确认**，需实测）。需新增"50 字拆段 + 顺序播放"层。
- **音质**：官方未提供音色列表/采样率说明，**无童声**，**采样率未能确认**。

### 2.2 微信云开发 + 云函数中转

- **原理**：小程序 `wx.cloud.callFunction({name:'tts', data:{text}})` → 云函数（Node.js，密钥存环境变量）调上游 TTS → 音频经 `cloud.uploadFile` 写云存储返回 `fileID` → 前端走**现有** `wx.cloud.downloadFile` 三级缓存。云函数可自由访问外网，**无小程序域名白名单问题**（[云函数官方指引](https://developers.weixin.qq.com/miniprogram/dev/wxcloud/guide/functions.html)）。
- **资费**（[云开发计费说明](https://developers.weixin.qq.com/minigame/dev/wxcloud/billing/price.html)，页面更新 2025-02-19）：
  - 免费体验活动：未发布阶段免费；**小程序发布后环境到期时间变为上线后第 15 天**；活动截止 2026-12-31；
  - 基础套餐 **19.9 元/月**（原价 39，5 折长期有效）：20 万次调用/月、外网出流量 2GB、CDN 流量 5GB 等；超量 0.5 元/万次、0.8 元/GB。
  - 注意：项目现有图片/音频下载已在消耗该环境配额，**当前环境是否仍在免费期需在控制台确认，不能假设免费**。
- **限制**：无主体限制（个人可用，相对 WechatSI 的核心优势）；单函数并发 20；端侧调用云函数默认超时约 3 秒（TTS 中转通常 1~3 秒，够用）；`callFunction` 返回体大小上限官方条文**未能确认**（建议走 fileID 回传规避）。
- **合规**：密钥在云函数内，不落前端，满足硬约束；不采集个人信息。
- **稳定性**：微信官方服务；主要风险是成本政策（发布后 15 天免费期到期、活动 2026 年底截止）。
- **接入复杂度**：与现有架构契合度最高——已有 `wx.cloud.init` + 云存储 + `downloadFile` 三级缓存，只需新增一个云函数 + `WxTtsBackend` 的在线兜底分支，`TtsBackend` 接口零改动。但**云函数只是中转层，音频仍需路线②的上游 TTS 产生，成本叠加**。
- **音质**：取决于上游 TTS，中转层无损耗。

## 3. 路线二：第三方云服务 TTS

三家均无小程序端官方 SDK/插件，均为服务端 API，个人实名认证即可开通（无企业主体门槛），密钥必须经云函数/服务端中转。

| | 腾讯云 TTS | 阿里云智能语音交互 | 讯飞在线语音合成 |
|---|---|---|---|
| **免费额度** | **800 万字符**（基础+精品音色，3 个月有效、限领一次，用尽停服不自动转付费） | 新用户免费试用 3 个月（2 路并发；试用期内具体调用量上限**未能确认**） | 每日 500 次（官方 API 文档）；2 万次/3 个月试用包（仅第三方教程佐证，**未能确认**） |
| **超出单价** | 精品 0.3 元/万字符（后付费默认关闭）；资源包最低 0.15 元/万字符 | **按次计费** 3.5 元/千次起（梯度降至 1.2 元/千次），每次 ≤300 字符 | 付费价目**未能确认**（价格页 JS 渲染抓取失败） |
| **单次长度** | **150 汉字**（古诗长文需切分） | 300 字符 | 8000 字节（约 2000 汉字，最宽裕） |
| **接口形态** | HTTPS + JSON，音频 base64（**最简单**） | WebSocket / RESTful，二进制流 | 仅 WebSocket 流式分片（最复杂；旧 HTTP 接口已停） |
| **中文音色** | 30+，8k/16k/24k；**有童声**（智萌·男童 101015、智甜·女童 101016 等），情感模式含 `poetry`（古诗朗读加分） | 100+，8K~48K；**童声最多**（知贝、思彤、杰力豆等），支持 SSML 情感 | 100+ 但免费版仅基础发音人，**童声能否免费用未能确认** |
| **并发** | 默认 20 | 试用 2 路 | 免费版有流控，数值未公开 |

来源：[腾讯计费概述](https://cloud.tencent.com/document/product/1073/34112)（2026-04 更新）、[腾讯 TextToVoice API](https://cloud.tencent.com/document/api/1073/37995)、[腾讯音色列表](https://cloud.tencent.com/document/product/1073/92668)、[阿里计费说明](https://help.aliyun.com/zh/isi/product-overview/billing-10)（2026-06）、[阿里接口说明](https://help.aliyun.com/zh/isi/developer-reference/overview-of-speech-synthesis)、[讯飞 API 文档](https://www.xfyun.cn/doc/tts/online_tts/API.html)。

- **合规**：上行文本全部是内置教学内容（非 UGC、不含儿童个人信息），不触碰《儿童个人信息网络保护规定》核心义务；三家服务协议禁止合成违规内容，内置教材天然合规。合成音频长期缓存未发现禁止条款（**未逐条核实协议全文**，商用前建议通读）。前提：不把用户自由输入送去合成。
- **稳定性**：三家均为长期运营的官方商用服务，无停用先例；主要风险是**免费额度的可持续性**——"3 个月有效、限领一次"明示免费是拉新手段。这恰恰是把免费额度用在**构建期一次性合成**而非运行时的理由。
- **接入复杂度**：
  - **变体甲（推荐，离线预生成）**：仿 `scripts/genWordAudios.mjs` 写生成脚本，用免费包批量合成全部中文语料 → 上传云存储 → `audioMap` 从单词表扩展为"文本 → fileID"映射。现有缓存/播放代码几乎原样复用，**零运行时成本、零密钥风险**。
  - **变体乙（在线兜底）**：`speak()` 未命中时 `wx.cloud.callFunction('tts')` → 云函数合成上传返回 fileID → 走现有 L3 下载链路。播放层（`wx.createInnerAudioContext`）不动；需叠加云开发基础套餐成本。
- **阿里云按次计费对本项目不友好**：读一个汉字也算 1 次（3.5‰ 元），本项目大量短文本场景下腾讯按字符计费明显更优。

## 4. 路线三：免 Key 直连接口（有道 dictvoice 类）

**2026-07-19 当日实测**（中国大陆网络，curl 只读 GET）：

| 接口 | 实测结果 | 结论 |
|---|---|---|
| `dict.youdao.com/dictvoice?audio=你好&le=zh` | 200，audio/mpeg | 存活，但限极短文本 |
| dictvoice 5 汉字 | 200 | 上限边界 |
| dictvoice **6 汉字** / 带标点短句 / 拼音「bā」/ 算式 | **500 全部失败** | **超限，读不了古诗/题干/拼音** |
| 有道 fanyivoice（昔日任意文本接口） | 异常状态码 688，空响应 | **已失效** |
| 搜狗 synthesis（曾有"小女孩"音色） | 302 重定向 HTML | **已失效** |
| 百度 text2audio | 返回 `Invalid param pid` / `Not verified user` | **已需认证** |
| edge-tts（微软） | 音色列表端点存活，但合成走 WSS + 动态令牌 | 2024-10 起遭微软反制，社区反复失效（[issue #290](https://github.com/rany2/edge-tts/issues/290)） |

- **资费**：免费、无额度概念（本质是网页内部接口被"借用"）。有道官方渠道是有道智云（注册送 50 元体验金，TTS 单价未能确认，[产品页](https://ai.youdao.com/product-tts.s) 动态渲染未取到数字）。
- **限制**：dictvoice 中文 ≤5 字是致命伤；小程序侧 `wx.downloadFile` 路径需把 `dict.youdao.com` 配入 downloadFile 合法域名（域名已备案，技术上可配，但属灰色操作）；`InnerAudioContext.src` 直读路径大概率无需配域名（官方文档未明文，**需真机验证**）。
- **合规**：不上行用户数据；但绕过官方付费 API 大规模调用属未授权使用，违反服务条款可能性高，音频版权有瑕疵。
- **稳定性**：**无任何官方背书、无 SLA**；同类接口已批量死亡；小程序请求 referer 固定为 `servicewechat.com/{appid}/...`，特征明显，随时可被一键封禁。风险定性：不是"会不会失效"，而是"何时失效"。
- **音质**：dictvoice 中文仅单一词典女声（MP3 64kbps/22.05kHz 单声道，当日实测），无音色选择、无童声。
- **结论**：**运行时直连不可用**（能力 + 稳定性双硬伤）。项目现状（构建期用 dictvoice 生成英文单词）属于构建工具用途，不受影响；中文预生成应改用支持长文本的正规接口。

## 6. 推荐方案

### 首选：腾讯云 TTS 免费包 + 构建期离线预生成

理由：

1. 语料是构建期已知的有限集合（< 1 万字符，见 §1.2），800 万字符免费包绰绰有余；
2. 与现有架构复用度最高——`WxTtsBackend` 三级缓存零改动，只扩展音频映射表；
3. 运行时零成本、零密钥、零外部依赖，无免费额度过期/被封的运行时风险；
4. 腾讯按字符计费 + JSON/base64 接口 + 有童声（智萌/智甜）+ `poetry` 情感模式，最贴合儿童古诗场景；
5. 规避了 WechatSI 的主体限制和云函数 19.9 元/月的固定成本。

### 分步实施计划

1. 【人工】注册腾讯云账号并个人实名认证 → 开通语音合成 → 控制台领取免费包（800 万字符，3 个月有效，**限领一次**，建议在语料清单冻结后领取）。
2. 新增 `scripts/genZhAudios.mjs`（仿 `genWordAudios.mjs`）：从 `poems.ts`、`hanzi*.json`、`pinyin*.json`、`wordProblemLogic.ts`、几何模板、固定引导语枚举全部中文文本 → 调腾讯 TTS（音色智萌/智甜，诗歌用 `EmotionCategory: poetry`；单诗 > 150 字按行切分合成）→ 输出 MP3。
3. 【人工】上传 MP3 到云存储 `audio/zh/`，生成/更新文本→fileID 映射数据。
4. 扩展 `miniprogram/src/platform/tts.ts`：`audioMap` 查找键从英文单词扩展为任意文本（建议文本规范化后作 key 或 hash），L1/L2/L3 缓存逻辑不动。
5. 验证：`cd miniprogram && npm run build:weapp` 通过；根目录 `npm test` 全绿；微信开发者工具逐模块试听（古诗整首、拼音、汉字、应用题、几何题干）。
6. 【人工】开发者工具内真机预览确认发音与缓存生效。

### 演进兜底（非本次范围）

- 若未来引入动态随机题干（`genLogic` 类运行时新文本）：增加云函数在线合成兜底（路线 ①b+② 叠加，成本 19.9 元/月起），`speak()` 未命中预生成表时走云函数，合成结果仍落云存储复用三级缓存。
- 若小程序主体变更/确认为企业：可评估 WechatSI 做 ≤50 字短文本实时朗读（免费、零服务端），代价是无童声、3 小时链接、需拆段层。

### 不建议

- 运行时直连免 Key 接口（路线③）：能力不够且随时失效；
- 阿里云按次计费路线：短文本场景单价劣势明显；
