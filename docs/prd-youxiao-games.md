## 1. 产品目标

**一句话定位**：一款纯前端、可免费托管于 GitHub Pages 的幼升小游戏合集网页，通过改编"羊了个羊 / 2048 / 连连看"等流行小游戏，让 5–7 岁儿童在趣味互动中掌握幼升小核心知识点。

**核心目标（4 条）**
1. **知识覆盖**：覆盖数学（20 以内加减）、语文拼音、语文常用字、英语四大模块的核心知识点。
2. **趣味驱动**：借成熟玩法降低学习门槛、提升留存，让"玩"即"学"。
3. **零门槛部署**：纯前端 + GitHub Pages，家长/教师打开即用，无需安装、无后端依赖。
4. **多端适配**：桌面与移动端响应式布局 + 触摸优化，随时随地可玩。

## 2. 用户故事

**儿童视角**
1. 作为小朋友，我希望用"羊了个羊"式的凑十法游戏把数字配对消除，以便在玩中学会 20 以内加法。
2. 作为小朋友，我希望在连连看里把加减法算式和答案配对消除，以便快速记住计算答案。
3. 作为小朋友，我希望拼音游戏里有声母、韵母配对与拼读，以便学会拼音拼读。
4. 作为小朋友，我希望翻牌记忆常用汉字并配合拼音，以便记住一二年级常见字。
5. 作为小朋友，我希望把英语 26 个字母的大小写拖拽配对，以便认识所有字母。
6. 作为小朋友，我希望用图文匹配认单词、用填空练简单句，以便积累英语词汇。
7. 作为小朋友，我希望每次答对都有音效和过关动画，以便获得成就感。
8. 作为小朋友，我希望在主页自由切换不同知识点的游戏，以便想玩什么就玩什么。

**家长视角**
9. 作为家长，我希望孩子打开网页就能玩、无需下载安装，以便利用碎片时间学习。
10. 作为家长，我希望有计分和过关反馈，以便直观了解孩子的学习进度。

**教师视角**
11. 作为幼教老师，我希望能在课堂用大屏/投影带孩子集体玩游戏，以便辅助教学。
12. 作为教师，我希望游戏难度可分级调节，以便适配不同水平的孩子（P2）。

## 3. 需求池（P0 / P1 / P2）

优先级说明：**P0** 必须首版交付 ｜ **P1** 应首版覆盖更多玩法 ｜ **P2** 后续迭代增强

| 编号 | 模块 | 需求描述 | 优先级 | 验收标准 |
|------|------|----------|--------|----------|
| G-001 | 主页 | 游戏选择主页，含 4 大模块入口（数学 / 语文拼音 / 语文常用字 / 英语），可点击进入 | P0 | 桌面与移动端均正常展示，4 个入口均可跳转至对应游戏 |
| G-002 | 数学 | 凑十法游戏（羊了个羊玩法）：将凑十的数字配对消除 | P0 | 正确判定凑十配对、消除后计分、完成关卡进入下一关 |
| G-003 | 数学 | 加减法连连看：算式与答案配对消除 | P0 | 配对正确即消除并计分，无可消除项时判定过关 |
| G-004 | 语文拼音 | 声母 / 韵母配对或拼读消除游戏（至少 1 个） | P0 | 覆盖常用声母韵母，配对正确有音效 / 视觉反馈 |
| G-005 | 语文常用字 | 翻牌记忆 / 连线匹配常用汉字（配合拼音，至少 1 个） | P0 | 汉字来自一二年级常见字表，可配对或记忆并计分 |
| G-006 | 英语 | 26 字母大小写配对游戏（至少 1 个） | P0 | 26 个字母均可配对，正确有反馈，全部完成即过关 |
| G-007 | 通用 | 计分 / 过关 / 音效反馈机制（每个游戏内置） | P0 | 每游戏有分数展示、过关判定、音效开关与播放 |
| G-008 | 通用 | 响应式布局 + 触摸操作适配（桌面 / 移动端） | P0 | 移动端触摸可正常操作，无布局错位、无溢出 |
| M-001 | 数学 | 数字合成 / 2048 式加法游戏 | P1 | 滑动合并同类数字，合并正确计分、可过关 |
| M-002 | 语文拼音 | 更多拼读组合玩法变体 | P1 | 覆盖更多声母+韵母拼读组合 |
| M-003 | 语文常用字 | 连线匹配 / 更多记忆玩法变体 | P1 | 多种玩法覆盖更多常见汉字 |
| M-004 | 英语 | 常用单词图文匹配 | P1 | 图片与单词可正确匹配并计分 |
| M-005 | 英语 | 日常简单句填空 | P1 | 可拖拽 / 选择填空，判定正确并反馈 |
| M-006 | 通用 | 答对攻击类玩法（如格斗答对攻击） | P1 | 答对触发攻击动画、答错受击，含计分与过关 |
| A-001 | 通用 | 成就系统（徽章 / 星星收集） | P2 | 完成目标解锁成就并在主页展示 |
| A-002 | 通用 | 难度分级（简单 / 中等 / 困难） | P2 | 可按知识点设置难度并影响题目 |
| A-003 | 家长 | 家长报告（学习时长 / 正确率） | P2 | 可查看 / 导出简单学习报告 |
| A-004 | 通用 | 动画增强（过关动画 / 角色动效） | P2 | 关键节点有动画反馈 |
| A-005 | 通用 | 本地存档（localStorage 记录进度） | P2 | 刷新后保留分数、星星与最高记录 |

### 4.3 视觉风格定义

- **主色（马卡龙柔色）**：蜜桃粉 `#FFB3C6`、薄荷绿 `#95E1C9`、天空蓝 `#A0D2FF`、柠檬黄 `#FFE066`
- **背景**：奶油白 `#FFF9F0`
- **字体**：圆体 / 卡通风格（推荐 "Baloo 2" / "Comic Sans MS" / 站酷快乐体），大字号、粗圆角
- **按钮 / 卡片**：大圆角（≥16px）、轻微投影、按压回弹动效
- **图标**：emoji 或扁平卡通图标，低龄友好、高对比度

## 5. 待确认问题（含默认建议）

1. **是否需要家长报告 + 本地存档？**
   默认建议：首版用 `localStorage` 存最高分与星星（A-005 纳入 P2 轻量实现）；家长报告（A-003）作为后续进阶，首版不做。
2. **知识点内容来源（汉字 / 拼音 / 单词表）由谁提供？**
   默认建议：内置教育部小学一、二年级常用字表 + 课标拼音表 + 基础英语词表，结构化为可配置 JSON，后续可替换。
3. **音效与背景音乐素材来源？**
   默认建议：使用 CC0 / 免费授权素材库（如 Pixabay、Freesound），避免版权风险；首版以短音效为主，暂不做背景音乐。
4. **成就系统与难度分级是否纳入首版？**
   默认建议：首版聚焦 P0 / P1；成就（A-001）、难度分级（A-002）作为 P2 后续迭代。
5. **是否需要在 GitHub Pages 之外提供 PWA 离线缓存？**
   默认建议：首版不做 PWA；纯静态托管即可，后续可加 Service Worker 支持离线。

## 6. 技术约束速览（供架构师参考）

- 纯前端实现，支持 GitHub Pages 部署，无后端
- 技术栈默认：Vite + React + MUI + Tailwind CSS（若无需框架，可降级为原生 HTML/CSS/JS 以减小体积）
- 响应式 + 触摸优先；状态用 localStorage 持久化
- 内容（题目/字词表）与逻辑解耦，以 JSON 配置驱动
